# SA-BNet 超参数优化建议

> 基于当前最优实验（bs=16, mIoU=94.66%）与本次探索实验（bs=32, 5 epoch）分析  
> 优先级分为 🔴 高 / 🟡 中 / 🟢 低

---

## 1. 🔴 学习率 & 调度策略（最高优先级）

### 1.1 学习率数值
| 策略 | 当前值 | 建议值 | 预期收益 |
|---|---|---|---|
| 初始 LR (classifier) | 0.01 | **0.005 ~ 0.02** | 更平稳的初期收敛 |
| 初始 LR (backbone) | 0.001 | **0.0005 ~ 0.002** | 保护预训练特征 |
| Warm-up 步数 | 无 | **200~500 iter** | 防止冷启动震荡 |

**分析**：目前使用固定比率 `backbone_lr = 0.1 × lr`，该比例已被广泛验证有效。但对于数据集较小（513张）的场景，降低初始 LR（如 0.005）往往能获得更稳定的收敛，减少 Abuth 类别的方差震荡。

**推荐实验**：
```bash
# 方案A：降低初始LR
--lr 0.005

# 方案B：加入学习率预热（需在main.py添加warmup支持）
--lr 0.01 --warmup_iters 300
```

### 1.2 学习率调度策略
| 策略 | 说明 | 适用场景 |
|---|---|---|
| **Poly**（当前） | 平滑衰减，大量语义分割任务的默认选择 | ✅ 保留，适合长训练 |
| **CosineAnnealing** | 带重启，防止局部最优 | 训练 >2000 iter 时值得尝试 |
| **OneCycleLR** | 先升后降，加速收敛 | bs=32 时较为有效 |

**建议**：在当前 Poly 策略基础上，探索 `CosineAnnealingLR`：
```python
scheduler = torch.optim.lr_scheduler.CosineAnnealingLR(
    optimizer, T_max=total_itrs, eta_min=1e-5
)
```

## 2. 🔴 Batch Size 与梯度噪声

| 配置 | bs | LR | 特点 |
|---|---|---|---|
| 基线最优 | 16 | 0.01 | 梯度噪声较大，有正则化效果 |
| 本次实验 | 32 | 0.02 | 梯度更稳定，收敛速度更快 |
| 建议探索 | **64** | **0.04** | 需验证显存（32GB 应可支持） |

**注意**：
- bs=32 已验证可行，在全量训练中很可能超过 bs=16 基线。
- bs>32 时，建议配合 **Gradient Clipping**（`max_norm=1.0`）防止梯度爆炸。
- 验证 batch 建议保持 4，避免大图（4080×3060）内存溢出（已在本次实验中验证）。

## 3. 🟡 边界损失超参（λ_edge & boundary_width）

### 3.1 边界损失权重 λ_edge
```
当前值：0.1
Loss 占比分析（Epoch5末）：seg_loss ≈ 0.163, edge_loss ≈ 0.125
实际边界贡献：0.1 × 0.125 = 0.0125 （约占 total loss 的 7%）
```

| λ_edge | 特点 | 建议场景 |
|---|---|---|
| 0.05 | 弱边界约束，更关注语义分割 | Cotton/Abuth 形状差异大时 |
| **0.1**（当前） | 平衡值 | ✅ 当前最优配置保留 |
| 0.2 | 强边界约束，有助于边缘精细化 | 样本边界模糊、类内混淆严重时 |
| 0.3~0.5 | 可能过度约束，反而降低 mIoU | 一般不推荐 |

**建议**：当 abuth↔cotton 混淆率 >20% 时，尝试提高 λ_edge 至 **0.2**；混淆率稳定后降回 0.1。

### 3.2 边界宽度 boundary_width
```
当前值：5 px（相对于 4080px 原图，约 0.1%）
```

| boundary_width | 效果 |
|---|---|
| 3 px | 更精细的边界监督，噪声更大 |
| **5 px**（当前） | ✅ 平衡噪声与精度 |
| 8 px | 更宽松的边界区域，适合边缘不清晰数据集 |

**建议**：可探索 `boundary_width=3` 与 `7` 的对比实验，对边界精度 F1 影响较大。

## 4. 🟡 权重衰减（Weight Decay）

```
当前值：1e-4
```

| weight_decay | 说明 |
|---|---|
| 1e-5 | 弱正则，适合数据量大时 |
| **1e-4**（当前） | ✅ 标准值，适合小数据集 |
| 5e-4 | 强正则，可缓解 513 张小样本过拟合 |

**建议**：鉴于训练集仅 513 张，可尝试 `weight_decay=5e-4` 以增强泛化能力，配合数据增强效果更佳。

## 5. 🟡 数据增强策略

**当前增强**（来自 `main.py`）：随机水平翻转、随机缩放裁剪、ColorJitter、归一化

| 增强方法 | 当前 | 建议加入 | 预期收益 |
|---|---|---|---|
| RandomHorizontalFlip | ✅ | — | — |
| RandomScale + Crop | ✅ | — | — |
| ColorJitter | ✅ | — | — |
| **RandomVerticalFlip** | ❌ | ✅ 推荐 | 农业图像上下对称性强 |
| **RandomRotation (±30°)** | ❌ | ✅ 推荐 | 提高叶片姿态鲁棒性 |
| **CutMix / MixUp** | ❌ | 🔬 可探索 | 缓解类别不平衡 |
| **Gaussian Noise** | ❌ | 🔬 可探索 | 提高光照鲁棒性 |

## 6. 🟡 ASPP 空洞率配置

```python
# 当前默认
dense_rates = (1, 3, 6, 12, 18)

# 建议对比实验
dense_rates = (1, 6, 12, 18, 24)   # 更大感受野，适合大尺寸目标
dense_rates = (1, 2, 4, 8, 16)     # 密集小步长，适合细粒度纹理
```

**分析**：cotton 与 abuth 在叶片纹理上差异主要在中等尺度，空洞率 `{6, 12}` 的覆盖对当前任务最关键。可实验验证不同配置的影响。

## 8. 🟢 Early Stop 配置

```
当前：patience=5, min_delta=0.0005, min_itrs=3000
```

| 参数 | 当前 | 建议 | 说明 |
|---|---|---|---|
| patience | 5 | **8~10** | 小数据集 mIoU 波动较大，patience 太小易提前停止 |
| min_delta | 0.0005 | **0.001** | 减少噪声触发的早停 |
| min_itrs | 3000 | **2000** | bs=32 时收敛更快，可适当提前允许早停 |

## 9. 综合优先实验路线

按投入产出比排序的建议实验顺序：

```
第1步：确认 bs=32 全量训练（50~100 epoch）基线
        --batch_size 32 --lr 0.02 --total_itrs 1600

第2步：数据增强扩充（vertical flip + rotation）
        在 main.py ExtRandomHorizontalFlip 旁加入 ExtRandomRotation

第3步：LR 对比（0.005 vs 0.01 vs 0.02）
        3组实验各跑 1600 iter，选最优 lr

第4步：λ_edge 调整（0.1 → 0.2）
        重点观察 abuth↔cotton 混淆率变化

第5步：weight_decay 增强（1e-4 → 5e-4）
        观察验证集泛化能力（val vs train mIoU 差距）
```
